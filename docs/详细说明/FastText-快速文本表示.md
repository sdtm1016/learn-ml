# FastText（快速文本表示）

## 简介
FastText由Facebook AI Research（FAIR）于2016年提出，是Word2Vec的改进版本。FastText通过引入子词（subword）信息，将每个词表示为其字符n-gram向量的总和，从而能够处理未登录词（OOV）问题，同时保持Word2Vec的训练速度优势。

## 核心原理

### 子词表示
- 将每个词分解为字符n-gram序列
- 每个n-gram有一个独立的向量
- 词的最终向量 = 词本身向量 + 所有n-gram向量之和

### 字符n-gram示例
```
"脑卒中" 的3-gram: <脑, 脑卒, 卒中, 中>
"where" 的3-gram: <wh, whe, her, ere, re>
```
- 添加特殊边界符号< >标记词的起止

### Skip-gram + 子词
```
中心词 w_t → 词向量 + Σ(n-gram向量) → 隐藏层 → 上下文词预测
```

## 数学公式

### 词的表示
```
v_w = v_word_w + Σ_{g ∈ G(w)} v_g
```
- v_word_w：词w本身的向量
- G(w)：词w的所有字符n-gram集合
- v_g：n-gram g的向量

### Skip-gram损失
```
L = -Σ_{c ∈ context} log σ(v_c · v_w) - Σ_{n=1}^{K} log σ(-v_n · v_w)
```

### 负采样
```
P(w) = f(w)^0.75 / Σ_{w'} f(w')^0.75
```

## 关键参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| dim | 词向量维度 | 100-300 |
| minn | 最小n-gram长度 | 3 |
| maxn | 最大n-gram长度 | 6 |
| ws | 上下文窗口大小 | 5 |
| neg | 负样本数量 | 5-20 |
| epoch | 训练轮数 | 5-25 |

## 医疗应用场景

### 医学未登录词处理
- 新发现的疾病名称表示
- 新药名称向量化
- 医学缩写展开和表示
- 复合医学术语分解

### 医疗文本处理
- 电子病历文本分类
- 医学文献相似度计算
- 医学实体识别特征
- 医疗问答系统

### 脑卒中专病数据采集
- 脑卒中相关术语向量化
- 处理非标准医学表达
- 医学缩写词表示
- 复合术语分解和理解

## 优势
- 支持未登录词（OOV）处理
- 训练速度快（与Word2Vec相当）
- 利用字符级信息增强表示
- 对形态学丰富的语言效果好
- 适合中文（汉字级别n-gram）
- 可处理词缀和构词法

## 局限性
- 静态词向量（一词一义）
- 无法处理多义词
- n-gram数量增加存储和计算开销
- 对低频词的子词信息可能不足
- 向量维度和Word2Vec一样固定

## 实现框架
- FastText官方库（C++/Python）
- Gensim：FastText封装
- PyTorch：自定义实现

## 与其他词向量对比

| 模型 | 上下文 | OOV处理 | 多义性 | 训练速度 |
|------|--------|---------|--------|---------|
| Word2Vec | 静态 | 不支持 | 不支持 | 快 |
| **FastText** | **静态** | **支持** | **不支持** | **快** |
| GloVe | 静态 | 不支持 | 不支持 | 快 |
| ELMo | 动态 | 部分支持 | 支持 | 中 |
| BERT | 动态 | 支持 | 支持 | 慢 |
