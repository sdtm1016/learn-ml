# EAST（高效准确的场景文本检测）

## 简介
EAST（Efficient and Accurate Scene Text Detector）是一种高效准确的场景文本检测算法，由华中科技大学提出。EAST使用全卷积网络直接预测文本区域，无需候选框生成和复杂的后处理步骤，在保持高精度的同时大幅提升了检测速度。

## 模型架构
- **骨干网络**：PVANet/ResNet/VGG特征提取
- **特征合并（Feature Merge）**：多尺度特征逐层合并
- **输出层**：预测文本得分图和几何形状

```
图像 → 骨干网络 → 特征合并 → 输出层 → 得分图 + 几何形状 → NMS → 文本框
```

## 核心原理

### 全卷积网络
- 完全基于卷积操作，不使用RNN或候选区域
- 输出与输入尺寸成比例的特征图
- 每个像素预测该位置是否为文本及其几何形状

### 特征合并
- 将不同尺度的特征图逐步合并
- Branch 1：合并conv3/4特征（小文本检测）
- Branch 2：合并conv4/5特征（中等文本检测）
- Branch 3：合并conv5/fc特征（大文本检测）
- 最终合并所有分支

### 几何形状预测
- **RBOX**：旋转矩形（x, y, w, h, θ）
- **QUAD**：四边形（8个坐标值）
- 得分图 + 几何形状共同确定文本区域

## 数学公式

### 得分图
```
S_{i,j} ∈ [0, 1]    # 文本区域置信度
```

### RBOX几何形状
```
R_{i,j} = (x, y, w, h, θ)    # 旋转矩形参数
```

### QUAD几何形状
```
Q_{i,j} = (Δx_1, Δy_1, ..., Δx_4, Δy_4)    # 四边形偏移量
```

### 损失函数
```
L = L_s + λ_g * L_g
```
- L_s：得分图损失（类别平衡交叉熵）
- L_g：几何形状损失（IoU损失）
- λ_g：平衡系数

### IoU损失
```
L_g = 1 - IoU(pred, GT)
```

## 后处理
- 将得分图阈值化
- 从预测的几何形状恢复文本框
- 局部感知NMS（Locality-Aware NMS）

## 医疗应用场景

### 病历扫描件检测
- 纸质病历文本区域定位
- 检查报告表头检测
- 处方单文本区域识别

### 医疗文档分析
- 医学影像中的文字标注检测
- 表格区域检测
- 医疗文档版面分析

### 脑卒中专病数据采集
- 扫描版脑卒中病历文本区域检测
- 影像报告文本定位
- 手写病历预处理

## 优势
- 检测速度快（GPU上实时检测）
- 全卷积网络，结构简洁
- 无需候选框生成
- 支持旋转文本检测
- 多尺度特征融合

## 局限性
- 不支持弯曲文本检测
- 极小文本检测效果一般
- 密集文本场景容易漏检
- 对不规则排列文本效果有限

## 实现框架
- PaddleOCR
- mmocr
- OpenCV DNN
