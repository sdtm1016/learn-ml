# Hierarchical-BERT（层次化BERT文本分类）

## 简介
Hierarchical-BERT是一种层次化文本分类模型，结合了文档级别的层次结构和BERT的强大编码能力。通过"词-句-文档"的层次化处理，有效解决长文档分类问题。对于超长文本（超过BERT 512 token限制），层次化方法能更好地捕获文档结构和语义信息。

## 模型架构
- **句子级BERT编码器**：对每个句子独立进行BERT编码
- **句子表示层**：将每个句子编码为固定维度向量
- **文档级编码器**：对句子序列进行编码，得到文档表示
- **分类层**：全连接分类输出

```
文档 → 分句 → 每句BERT编码 → 句子向量 → 文档级编码 → 分类
              (词-句层次)                  (句-文档层次)
```

## 核心原理

### 层次化结构
1. **词级别**：BERT对每个句子内的词进行编码
2. **句级别**：聚合句子内的词向量得到句子表示
3. **文档级别**：聚合句子向量得到文档表示

### 句子表示方法
- **[CLS]池化**：取每句BERT的[CLS]输出
- **平均池化**：对所有token输出取平均
- **最大池化**：取所有token输出的最大值

### 文档级编码方法
- **BiLSTM**：对句子序列进行双向LSTM编码
- **Attention**：注意力加权聚合句子向量
- **HAN（层次注意力网络）**：词级+句级双层注意力

## 数学公式

### 句子编码
```
h_i = BERT(s_i)[CLS]    # 第i个句子的BERT [CLS]表示
```

### 文档编码（BiLSTM方式）
```
H = BiLSTM([h_1, h_2, ..., h_n])
s_doc = H[-1]           # 最后时刻的隐藏状态
```

### 文档编码（Attention方式）
```
u_i = tanh(W_s · h_i + b_s)
α_i = softmax(u_i^T · u_w)
s_doc = Σ_i α_i · h_i
```

### 分类
```
y = Softmax(W_o · s_doc + b_o)
```

## 医疗应用场景

### 长病历文档分类
- 完整病历文档分类（入院记录/出院小结/手术记录）
- 多段落临床笔记分类
- 长篇检查报告分类

### 医学文献分类
- 医学论文主题分类
- 研究类型识别（RCT/队列/病例报告）
- 循证医学等级分类

### 脑卒中专病数据采集
- 完整脑卒中病历文档分类
- 多段落病历信息类型识别
- 脑卒中研究报告分类
- 长篇影像报告分类

### 临床研究文档处理
- 临床试验方案文档分类
- 研究报告章节识别
- 不良事件报告分类

## 优势
- 有效处理超长文档（突破512 token限制）
- 保留文档的层次化结构信息
- 词-句-文档的多粒度理解
- 每层可独立微调
- 灵活的文档级编码器选择

## 局限性
- 计算复杂度高（每个句子都要过BERT）
- 训练和推理速度慢
- 句子间的上下文可能丢失（独立编码）
- 显存消耗大
- 对短文本效果提升不明显

## 变体

| 变体 | 文档编码器 | 特点 |
|------|-----------|------|
| HAN-BERT | 层次注意力 | 双层注意力，可解释 |
| BiLSTM-BERT | BiLSTM | 捕获句子间序列关系 |
| Transformer-BERT | Transformer | 句子间自注意力 |
| BERT-over-BERT | BERT | 上下文跨句子建模 |

## 实现框架
- PyTorch + Hugging Face Transformers
- PyTorch Lightning
