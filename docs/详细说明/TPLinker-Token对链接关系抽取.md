# TPLinker（Token Pair Linking关系抽取）

## 简介
TPLinker（Token Pair Linking）是一种基于Token对链接的单阶段联合实体关系抽取模型，通过建立token对之间的链接关系来实现实体和关系的联合抽取。由南京大学提出，能够高效处理重叠三元组问题。

## 模型架构
- **BERT编码层**：获取上下文感知的文本表示
- **Token Pair表示层**：构建token对的特征表示
- **握手标注层**：通过token对之间的链接关系标注实体和关系

```
输入文本 → BERT编码器 → Token对特征构建 → 握手标注 → 三元组集合
```

## 核心原理

### Token Pair Linking
- 将实体识别和关系抽取统一为token对之间的链接问题
- **Entity-to-Entity Linking**：标记实体首尾token的链接
- **Subject-to-Object Linking**：标记主体首token到客体首token的链接
- **Subject-to-Object Linking**：标记主体首token到客体尾token的链接

### 握手标注（Handshaking Tagging）
- 将所有token的两两配对，形成token pair矩阵
- 通过预测token pair之间的链接关系实现实体和关系标注
- 单阶段完成实体和关系的联合抽取

### 矩阵构建
- 输入长度为N的序列
- 构建N×N的token pair矩阵
- 每个token pair预测多种标签（实体头尾、关系头尾等）

## 数学公式

### Token Pair特征
```
h_{i,j} = [h_i; h_j; h_i ⊙ h_j; PE(i,j)]
```
- h_i, h_j：BERT输出的token向量
- ⊙：逐元素乘法
- PE(i,j)：位置编码

### 标签矩阵
- EH-to-ET矩阵：实体头到实体尾的链接
- SH-to-OH矩阵：主体头到客体头的链接
- SH-to-OT矩阵：主体头到客体尾的链接

### 损失函数
```
L = Σ_{matrix} BCELoss(y_matrix, y_pred_matrix)
```

## 医疗应用场景

### 电子病历信息抽取
- 联合抽取疾病-症状-治疗关系
- 重叠关系处理（同一疾病多个症状）
- 病历结构化信息提取

### 药物知识图谱构建
- 药物多适应症关系抽取
- 药物-药物相互作用
- 药物-不良反应关系

### 脑卒中专病数据采集
- 脑卒中多维度关系抽取
- 危险因素-疾病-治疗链条
- 影像-诊断-治疗方案关联

## 优势
- 单阶段联合抽取，无需级联
- 高效处理重叠三元组
- 统一框架处理实体和关系
- 训练效率高
- 避免曝光偏差问题

## 局限性
- Token pair矩阵空间复杂度为O(N²)
- 长文本处理时内存消耗大
- 需要大量训练数据
- 关系类型多时标签矩阵维度大
