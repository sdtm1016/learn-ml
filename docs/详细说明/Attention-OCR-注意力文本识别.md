# Attention-OCR（注意力机制文本识别）

## 简介
Attention-OCR是一种基于注意力机制的文本识别模型，通过注意力机制将CNN提取的视觉特征与RNN的解码过程对齐，实现了更精细的字符级识别。相比CTC方法，注意力机制能够显式建模字符与图像区域之间的对应关系。

## 模型架构
- **CNN编码器**：提取图像视觉特征
- **注意力RNN解码器**：带注意力的序列解码
- **字符输出**：逐字符生成识别结果

```
图像 → CNN编码器 → 特征序列 → 注意力RNN解码器 → 字符序列
                                ↑ 注意力对齐
```

## 核心原理

### CNN编码
- 使用CNN（如VGG/ResNet）提取图像特征
- 输出特征图作为编码器状态
- 每个空间位置对应一个特征向量

### 注意力机制
- 在每步解码时，计算注意力权重
- 权重反映当前生成字符应关注的图像区域
- 加权求和得到当前步骤的上下文向量

### RNN解码
- 带注意力的GRU/LSTM解码器
- 每步生成一个字符
- 利用前一步的输出和注意力上下文

## 数学公式

### 编码器
```
H = CNN(image)    # H: H' × W' × D
```

### 注意力计算
```
e_{t,i} = a(h_{t-1}, h_i)        # 注意力得分
α_{t,i} = softmax(e_{t,i})        # 注意力权重
c_t = Σ_i α_{t,i} · h_i          # 上下文向量
```

### 解码器
```
s_t = RNN(y_{t-1}, s_{t-1}, c_t)    # 隐藏状态
y_t = Softmax(W · s_t + b)           # 字符概率
```

### 损失函数
```
L = -Σ_t log P(y_t | y_{<t}, image)
```

## 注意力机制类型

| 类型 | 公式 | 特点 |
|------|------|------|
| 加性注意力 | a = v^T tanh(W_h h + W_s s) | 经典方法 |
| 乘性注意力 | a = s^T W h | 计算高效 |
| 覆盖注意力 | a = a + coverage | 防止重复关注 |

## 医疗应用场景

### 精细OCR识别
- 医学文档中密集文本识别
- 处方药物名称精确识别
- 检查报告数值精确识别
- 病理报告文本识别

### 医疗影像标注读取
- 医学影像上的文字标注读取
- 图像测量值识别
- 患者信息标注提取

### 脑卒中专病数据采集
- 脑卒中病历精细OCR
- 影像报告数值精确提取
- 检验结果OCR识别
- 手写病历识别

## 优势
- 显式建模字符-图像区域对齐关系
- 注意力权重可解释（可视化关注区域）
- 无需CTC的独立性假设
- 支持不定长文本识别
- 识别精度高

## 局限性
- 训练速度较慢（逐步解码）
- 推理速度比CTC慢
- 容易出现注意力漂移
- 长文本序列注意力不稳定
- 对弯曲/变形文本支持有限

## 实现框架
- PyTorch：自定义Attention + GRU/LSTM
- TensorFlow：tf.keras Attention
- PaddleOCR
- mmocr

## 与其他模型对比

| 模型 | 解码方式 | 准确率 | 可解释性 | 速度 |
|------|---------|--------|---------|------|
| **CRNN** | **CTC** | 高 | 低 | 快 |
| **Attention-OCR** | **注意力** | 高 | **高** | 中 |
| ASTER | 注意力+STN | 高 | 高 | 中 |
| SAR | 注意力+语义增强 | 更高 | 高 | 中 |
| TrOCR | Transformer注意力 | 最高 | 高 | 慢 |
