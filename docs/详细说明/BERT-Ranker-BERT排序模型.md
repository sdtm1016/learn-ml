# BERT-Ranker（BERT排序模型）

## 简介
BERT-Ranker是一种基于BERT预训练模型的排序/重排模型，用于对候选实体进行相关性排序。在实体链接、语义匹配、信息检索等任务中，BERT-Ranker通过BERT编码实体-候选对的语义信息，计算匹配分数，对候选结果进行排序。

## 模型架构
- **BERT编码器**：编码实体提及和候选实体的上下文
- **交互/拼接层**：处理实体和候选的交互特征
- **分类/打分层**：输出匹配分数或分类结果

```
实体提及 + 候选实体 → BERT编码器 → [CLS]表示 → 全连接层 → 匹配分数
```

## 核心原理

### BERT语义编码
- 将实体提及及其上下文和候选实体拼接为输入
- BERT编码器学习两者之间的语义关联
- [CLS]向量聚合交互信息

### 候选排序
- 对每个候选实体计算匹配分数
- 按分数降序排列
- 选择Top-K作为链接结果

### 输入格式
```
[CLS] 实体提及上下文 [SEP] 候选实体描述 [SEP]
```

## 数学公式

### BERT编码
```
H = BERT([CLS] context [SEP] candidate [SEP])
h_cls = H[0]
```

### 匹配分数
```
score = sigmoid(W · h_cls + b)
```

### 排序损失
```
L = -log σ(score_positive - score_negative + margin)
```

## 医疗应用场景

### 实体链接与标准化
- 医学实体到标准术语的链接
- 疾病名称到ICD-10编码的映射
- 药物名称到标准药物库的匹配

### 术语归一化
- 同义词归一化（如"脑梗"→"脑梗死"）
- 缩写展开（如"CT"→"计算机断层扫描"）
- 非标准术语标准化

### 脑卒中专病数据采集
- 脑卒中相关实体标准化
- 医学术语到标准编码的映射
- 多种表达方式的术语统一
- 知识库实体匹配

## 优势
- 强大的语义理解能力
- 可利用预训练知识
- 对上下文敏感
- 适用于精细排序
- 支持多种排序任务

## 局限性
- 每个候选需要一次BERT前向计算
- 候选多时计算开销大
- 推理速度慢
- 需要负样本训练

## 与Bi-Encoder/Cross-Encoder的关系

| 类型 | 编码方式 | 交互方式 | 速度 | 精度 |
|------|---------|---------|------|------|
| Bi-Encoder | 分别编码 | 向量相似度 | 快 | 中 |
| **BERT-Ranker (Cross-Encoder)** | **拼接编码** | **深层交互** | 慢 | **高** |

## 实现框架
- Hugging Face Transformers
- sentence-transformers
- PyTorch
- Faiss（向量检索加速）

## 排序策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| Pointwise | 每个候选独立打分 | 简单排序 |
| Pairwise | 候选对比较 | 精细排序 |
| Listwise | 候选列表整体排序 | 搜索排序 |
