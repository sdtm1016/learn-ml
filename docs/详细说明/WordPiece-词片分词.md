# WordPiece（词片分词）

## 简介
WordPiece是一种子词分词算法，由Google团队开发，最初用于日语和韩语语音搜索系统。WordPiece基于语言模型似然度来选择合并的子词，而非简单的频率统计，是BERT系列模型的标准分词方法。

## 核心原理

### 算法流程

**训练阶段（构建词表）**：
1. 初始化词表为所有单个字符
2. 统计语料中所有词的频率
3. 对每个可能的相邻符号对，计算合并后语言模型的似然度增益
4. 选择增益最大的符号对合并为新符号
5. 重复步骤3-4，直到达到目标词表大小

**编码阶段（分词）**：
1. 将输入词拆分为字符
2. 使用贪心最长匹配算法从词表中查找子词
3. 以"##"前缀标记非词首子词

### 示例
```
"脑卒中" → 拆分为字符 → "脑" "卒" "中"
可能的子词: "脑" "卒中" "中" → "脑" "卒" "中"
或合并后: "脑卒中"

"unaffable" → "un" "##aff" "##able"
"##"前缀表示该子词不是词的开头
```

## 数学公式

### 似然度增益
```
score(pair) = P(AB) / (P(A) × P(B))
```
- A, B：相邻的符号对
- AB：合并后的符号
- P(x)：符号x在语料中的频率概率
- 选择score最大的pair合并

### 语言模型似然度
```
L = Σ_{w ∈ vocab} count(w) × log P(w)
P(w) = Σ_{s ∈ splits(w)} Π_{t ∈ s} P(t)
```

### 编码（贪心最长匹配）
```
encode(word):
    start = 0
    while start < len(word):
        end = len(word)
        while end > start:
            substr = word[start:end]
            if start > 0:
                substr = "##" + substr
            if substr in vocab:
                tokens.append(substr)
                break
            end -= 1
        start = end
```

## 关键特性

| 特性 | 说明 |
|------|------|
| ##前缀 | 标记非词首子词 |
| 似然度选择 | 基于语言模型增益而非纯频率 |
| 贪心匹配 | 编码时使用最长匹配策略 |
| [UNK]标记 | 无法分解的词用[UNK]替代 |

## 医疗应用场景

### 医疗文本分词
- BERT系列模型的标准分词
- 医学实体子词切分
- 医学复合词处理

### 脑卒中专病数据采集
- 脑卒中相关文本分词
- BERT模型输入预处理
- 医学文本Token化

## 优势
- 基于语言模型似然度，更合理的合并策略
- 有效处理OOV问题
- 高频词保持完整
- 与BERT完美配合
- ##前缀清晰标记子词边界

## 局限性
- 训练速度慢（每次合并需计算似然度）
- 贪心匹配可能非最优
- 对非拉丁语系（如中文）需要特殊处理
- 词表固定后不易扩展
- 与BPE相比实现更复杂

## 实现框架
- Hugging Face tokenizers：BertWordPieceTokenizer
- Google BERT官方实现
- TensorFlow Text

## 与BPE对比

| 特性 | BPE | WordPiece |
|------|-----|-----------|
| 合并标准 | 频率最高 | 似然度增益最大 |
| 训练速度 | 快 | 慢 |
| 子词标记 | 无特殊标记 | ##前缀 |
| 使用模型 | GPT系列 | BERT系列 |
| 效果 | 相当 | 相当 |
