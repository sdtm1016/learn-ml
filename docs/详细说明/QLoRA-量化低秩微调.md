# QLoRA（量化低秩自适应微调）

## 简介
QLoRA（Quantized LoRA）是一种结合4-bit量化技术和LoRA微调的高效微调方法，由华盛顿大学于2023年提出。通过将预训练模型量化为4-bit NF4格式，配合LoRA适配器训练，使得在单张消费级GPU上微调65B参数大模型成为可能。

## 核心原理

### 4-bit NF4量化
- 将预训练模型权重从FP32/BF16量化为4-bit NF4（NormalFloat 4）
- NF4：一种信息论最优的数据类型，适合正态分布权重
- 量化大幅减少显存占用（约4x压缩）

### 双重量化（Double Quantization）
- 对量化常数（quantization constants）进行二次量化
- 进一步减少显存占用（每个参数节省约0.37 bit）
- 对分组大小64的量化常数再进行8-bit量化

### 分页优化器（Paged Optimizers）
- 使用NVIDIA统一内存管理
- 当GPU显存不足时，自动将优化器状态转移到CPU内存
- 避免显存溢出导致的训练中断

### LoRA适配器
- 在量化后的模型上添加LoRA适配器
- LoRA参数保持BF16精度训练
- 推理时可合并为无额外开销的模型

## 数学公式

### NF4量化
```
W_q = NF4_quantize(W_0)
W_dequant = NF4_dequantize(W_q)
```
- W_0：原始FP32权重
- W_q：4-bit量化权重
- 前向传播使用反量化后的权重

### 双重量化
```
c_q = quantize_8bit(quantization_constants)
c_dequant = dequant_8bit(c_q)
```

### QLoRA前向传播
```
h = (W_0_quantized + B·A) · x
```
- W_0_quantized：4-bit量化权重（冻结）
- B·A：BF16精度的LoRA适配器（训练）

### 损失函数
```
L = TaskLoss(f(x; W_0_q + (α/r)·B·A), y)
```

## 关键配置

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 量化位数 | 权重量化精度 | 4-bit (NF4) |
| 双重量化 | 量化常数再量化 | 启用 |
| 分页优化器 | 显存不足时转移到CPU | 启用 |
| LoRA秩 | 适配器秩 | 16-64 |
| LoRA目标 | 应用LoRA的模块 | q_proj, v_proj, k_proj, o_proj |
| LoRA Dropout | 适配器Dropout | 0.05 |
| 批大小 | 训练批大小 | 按显存调整 |

## 医疗应用场景

### 消费级GPU微调医疗大模型
- 单卡（24GB）微调7B-13B医疗模型
- 单卡（48GB）微调30B-65B医疗模型
- 低成本医疗大模型定制

### 脑卒中专病数据采集
- 脑卒中领域大模型低成本微调
- 多种医疗任务快速适配
- 病历结构化模型训练
- 医疗实体抽取模型微调

### 医疗AI普及
- 降低医疗AI开发门槛
- 基层医院本地部署大模型
- 医疗数据不出院的模型训练

## 优势
- 极低显存占用（65B模型可在48GB GPU微调）
- 保持全参数微调的性能
- 推理时无额外开销（可合并LoRA）
- 成本大幅降低
- 适合资源受限场景

## 局限性
- 训练速度较慢（反量化开销）
- 4-bit量化可能有微小精度损失
- 需要特定硬件支持（bfloat16）
- 对超长序列支持有限
- 量化过程需要额外预处理时间

## 实现框架
- bitsandbytes：量化后端
- PEFT（Hugging Face）：LoRA配置
- Transformers：模型加载
- Accelerate：分布式训练
