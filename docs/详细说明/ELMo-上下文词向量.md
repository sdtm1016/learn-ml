# ELMo（上下文词向量）

## 简介
ELMo（Embeddings from Language Models）由Allen AI于2018年提出，是一种基于双向LSTM语言模型的上下文词向量表示方法。与Word2Vec等静态词向量不同，ELMo为每个词根据其上下文生成不同的向量表示，能够自然地处理多义词问题。

## 模型架构
- **前向LSTM**：从左到右的语言模型
- **后向LSTM**：从右到左的语言模型
- **多层表示**：组合不同层的LSTM输出

```
输入层 → 前向LSTM（L层）→ 前向语言模型
输入层 → 后向LSTM（L层）→ 后向语言模型
                ↓
          加权组合各层输出 → 上下文词向量
```

## 核心原理

### 双向语言模型
- **前向LM**：P(w_1, ..., w_N) = Π P(w_t | w_1, ..., w_{t-1})
- **后向LM**：P(w_1, ..., w_N) = Π P(w_t | w_{t+1}, ..., w_N)
- 联合最大化双向语言模型

### 多层表示
- ELMo使用L层BiLSTM
- 不同层捕获不同级别的语言特征
- 底层：句法特征（词性、语法结构）
- 高层：语义特征（语义角色、情感）

### 上下文词向量
```
ELMo_k = γ · Σ_{j=0}^{L} s_j · h_{k,j}
```
- h_{k,j}：第k个token在第j层的BiLSTM输出
- s_j：可学习的层权重
- γ：可学习的缩放因子

## 数学公式

### 前向语言模型
```
h_{t,j} = LSTM_j(h_{t,j-1}, h_{t-1,j})
P(w_t | w_1, ..., w_{t-1}) = softmax(h_{t,L} · W_e)
```

### 后向语言模型
```
h_{t,j} = LSTM_j(h_{t,j-1}, h_{t+1,j})
P(w_t | w_{t+1}, ..., w_N) = softmax(h_{t,L} · W_e)
```

### 损失函数
```
L = Σ_{t=1}^{N} (log P(w_t | w_1, ..., w_{t-1}) + log P(w_t | w_{t+1}, ..., w_N))
```

### ELMo向量
```
ELMo_k = γ · Σ_{j=0}^{L} s_j · h_{k,j}
```
- h_{k,0}：词嵌入层
- h_{k,1}...h_{k,L}：各层BiLSTM输出

## 医疗应用场景

### 多义医学术语表示
- "中风"在不同上下文中的不同含义
- "阳性"在不同检查项目中的不同解释
- 医学缩写的上下文消歧

### 医疗文本处理
- 电子病历信息抽取
- 医学实体识别（利用上下文信息）
- 医学文本相似度计算
- 医疗问答系统

### 脑卒中专病数据采集
- 脑卒中术语上下文理解
- 多义医学概念消歧
- 病历文本上下文分析
- 医学实体关系抽取

## 优势
- 上下文感知，一词多义
- 多层表示融合不同级别特征
- 无需任务特定标注（无监督预训练）
- 可直接用于下游任务（加权组合）
- 对多义词处理效果好

## 局限性
- LSTM架构，训练和推理速度慢
- 无法并行计算（序列依赖）
- 长距离依赖能力有限
- 预训练语料有限（比BERT小）
- 表示能力不如Transformer-based模型
- 向量维度较大（1024维）

## 实现框架
- AllenNLP（官方实现）
- TensorFlow Hub
- PyTorch（自定义实现）
- allennlp

## 与其他词向量对比

| 模型 | 上下文 | 架构 | 多义性 | 训练速度 | 表示能力 |
|------|--------|------|--------|---------|---------|
| Word2Vec | 静态 | 浅层 | 不支持 | 最快 | 低 |
| FastText | 静态 | 浅层 | 不支持 | 最快 | 低 |
| **ELMo** | **动态** | **BiLSTM** | **支持** | 中 | 中 |
| BERT | 动态 | Transformer | 支持 | 慢 | 高 |
| GPT | 动态 | Transformer | 支持 | 慢 | 高 |
