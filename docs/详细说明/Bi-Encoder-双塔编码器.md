# Bi-Encoder（双塔编码器）

## 简介
Bi-Encoder（双塔编码器）是一种将两个文本分别通过独立的编码器编码为向量，然后通过向量相似度计算匹配分数的模型架构。Bi-Encoder支持将编码向量预先计算并索引，实现高效的向量检索，是语义搜索、实体链接等任务的主流方案。

## 模型架构
- **编码器A**：编码文本A为稠密向量
- **编码器B**：编码文本B为稠密向量（通常与A共享权重）
- **相似度计算**：计算两个向量的余弦相似度

```
文本A → 编码器 → 向量A ↘
                         余弦相似度 → 匹配分数
文本B → 编码器 → 向量B ↗
```

## 核心原理

### 独立编码
- 两个文本分别通过（共享权重的）编码器
- 每个文本编码为固定维度的稠密向量
- 向量表示文本的语义信息

### 向量相似度
```
sim(A, B) = cosine(e_A, e_B) = (e_A · e_B) / (||e_A|| · ||e_B||)
```
- 余弦相似度衡量两个向量的方向一致性
- 范围[-1, 1]，值越大表示越相似

### 预计算与索引
- 编码器输出可预先计算并存储
- 使用FAISS等向量索引实现毫秒级检索
- 适合大规模候选库

## 数学公式

### 编码
```
e_A = Encoder(A)    # A的向量表示
e_B = Encoder(B)    # B的向量表示
```

### 相似度
```
score = cosine(e_A, e_B) = (e_A · e_B) / (||e_A|| · ||e_B||)
```

### 训练损失（对比学习）
```
L = -log exp(sim(A, B+) / τ) / Σ_i exp(sim(A, B_i) / τ)
```
- B+：正样本
- B_i：所有样本（含正负样本）
- τ：温度参数

### 知识蒸馏损失
```
L = KL(CrossEncoder_score || BiEncoder_score)
```

## 与Cross-Encoder对比

| 特征 | Bi-Encoder | Cross-Encoder |
|------|-----------|--------------|
| 编码方式 | 分别独立编码 | 拼接后统一编码 |
| 交互方式 | 向量级相似度 | 全token交叉注意力 |
| 速度 | 快（预计算向量） | 慢（每对重新计算） |
| 精度 | 中 | 高 |
| 索引支持 | 支持（FAISS） | 不支持 |
| 适用场景 | 召回/初筛 | 精排/重排 |

## 医疗应用场景

### 实体链接召回
- 医学实体候选快速召回
- 从大规模术语库中检索候选
- ICD-10/SNOMED CT编码候选召回

### 语义搜索
- 病历文本语义搜索
- 医学文献检索
- 临床指南查找

### 脑卒中专病数据采集
- 脑卒中术语快速候选召回
- 标准术语库快速匹配
- 大规模医学知识库检索
- 同义词快速发现

### 问答系统
- 医学问答候选段落检索
- 知识库问答召回

## 典型使用流程

```
第一步（索引构建）：用Bi-Encoder编码所有术语库条目，构建FAISS索引
第二步（召回）：编码查询实体，在FAISS中检索Top-K候选
第三步（精排）：可选Cross-Encoder重排
```

## 优势
- 编码向量可预计算，检索速度快
- 支持大规模向量索引（FAISS）
- 适合召回/初筛阶段
- 训练和推理效率高
- 易于扩展新实体

## 局限性
- 缺乏深层token级别交互
- 匹配精度不如Cross-Encoder
- 编码器参数共享可能不适合所有场景
- 需要精心设计负样本训练策略
- 长文本编码可能丢失细节

## 实现框架
- sentence-transformers
- Hugging Face Transformers
- FAISS（向量索引）
- Milvus（向量数据库）
- FlagEmbedding
