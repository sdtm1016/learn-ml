# LoRA（低秩自适应微调）

## 简介
LoRA（Low-Rank Adaptation）是一种高效的参数高效微调方法，由微软于2021年提出。通过冻结预训练模型的原始权重，仅训练低秩分解矩阵来实现模型适配，大幅减少微调所需的参数量和显存占用，同时保持与全参数微调相当的性能。

## 核心原理

### 低秩假设
- 预训练模型权重更新矩阵ΔW具有低秩特性
- 可以分解为两个低秩矩阵的乘积
- 仅需训练极少参数即可实现有效微调

### 权重更新方式
```
W_new = W_0 + ΔW = W_0 + B · A
```
- W_0：原始预训练权重（冻结）
- B ∈ R^{d×r}：降维矩阵（训练）
- A ∈ R^{r×k}：升维矩阵（训练）
- r << min(d, k)：秩（超参数）

### 前向传播
```
h = W_0 · x + (B · A) · x = W_0 · x + B · (A · x)
```
- 可合并为单个矩阵实现零推理开销

## 数学公式

### 低秩分解
```
ΔW = B · A
B初始化为0，A初始化为高斯随机分布
```

### 缩放因子
```
h = W_0 · x + (α/r) · B · A · x
```
- α：缩放超参数
- r：秩
- α/r保持不同秩下的更新幅度一致

### 损失函数
```
L = TaskLoss(f(x; W_0 + (α/r)·B·A), y)
```

## 关键超参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| r（秩） | 低秩矩阵的维度 | 4, 8, 16, 32, 64 |
| α（缩放） | 缩放因子 | 16, 32 |
| 目标模块 | 应用LoRA的层 | q_proj, v_proj |
| Dropout | LoRA层Dropout | 0.05-0.1 |

## 医疗应用场景

### 医疗大模型微调
- 医疗LLM领域适配
- 电子病历理解模型微调
- 医学问答模型训练
- 临床决策支持模型

### 脑卒中专病数据采集
- 脑卒中领域大模型微调
- 医疗实体抽取模型适配
- 病历结构化模型训练
- 多任务学习

### 低资源场景
- 单GPU医疗模型微调
- 多个医疗任务切换（切换LoRA适配器）
- 医疗模型快速迭代

## 优势
- 参数量极少（<1%原始参数）
- 显存占用大幅降低
- 训练速度快
- 推理时无额外开销（可合并权重）
- 多任务适配器可热切换
- 不增加推理延迟

## 局限性
- 秩r的选择需要调优
- 对极复杂任务可能不如全参数微调
- 需要精心选择目标模块
- 合并权重后无法取消适配

## 实现框架
- PEFT（Hugging Face）：LoRA类
- LoRA官方仓库：microsoft/LoRA
- PyTorch：自定义实现
- DeepSpeed：集成LoRA
