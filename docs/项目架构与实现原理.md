# 机器学习算法学习平台 - 架构与实现原理

## 一、项目概述

### 1.1 项目定位
本项目是一个基于 React + TypeScript + Vite 构建的**交互式机器学习算法学习平台**，旨在为学习者提供：
- 系统化的算法知识库（100+ 算法文档）
- 可视化的算法关系图谱
- 手绘风格的原理图解（60+ 张 Excalidraw 图）
- 结构化的学习路径规划
- 进度跟踪与算法对比功能

### 1.2 核心价值
- **知识可视化**：将抽象的算法概念转化为直观的图形表达
- **学习路径化**：提供从入门到高级的结构化学习路线
- **交互体验**：模糊搜索、快捷键、主题切换等现代化交互
- **医疗场景聚焦**：每个算法都配有医疗领域的实际应用案例

## 三、核心设计原理

### 3.1 数据驱动架构

#### 算法数据模型

项目采用 **类型安全的数据驱动架构**，所有算法信息存储在 TypeScript 数据文件中：

```typescript
// 核心类型定义 (src/data/algorithms/types.ts)
export type AlgorithmItem = {
  name: string;              // 算法名称（唯一标识）
  label: string;             // 显示标签
  family: string;            // 算法家族（如"线性模型"）
  difficulty: '入门' | '进阶' | '高级';  // 难度级别
  categories: AlgorithmCategory[];       // 所属分类（多标签）
  description: string;       // 简短描述
  intuition: string;         // 核心直觉
  whenToUse: string[];       // 使用场景
  strengths: string[];       // 优势
  limitations: string[];     // 局限
  metrics: string;           // 评估指标
  plainExplanation: string;  // 通俗解释（自动生成）
  medicalExample: string;    // 医疗案例（自动生成）
  fallbackExample: string;   // 通用案例（自动生成）
  whyItFits: string;         // 适用性分析（自动生成）
  relatedDocumentId?: string;// 关联文档ID
  codeExamples?: {...};      // 代码示例链接
};
```

#### 分类体系

算法按 17 个维度分类，支持**多标签体系**：

- **基础类型**：监督学习、无监督学习、集成学习、深度学习
- **技术维度**：降维、异常检测、时间序列、NLP、视觉、图学习
- **应用领域**：生存分析、因果推断、推荐系统、强化学习
- **辅助类别**：数据预处理、模型评估

#### 叙事生成器（Narrative Builder）

**核心创新**：使用 `buildNarrative()` 函数自动生成教学叙事，避免手工维护重复内容：

```typescript
// src/data/algorithms/narrative.ts
export function buildNarrative(algorithm: BaseAlgorithmItem): NarrativeFields {
  return {
    plainExplanation: generatePlainExplanation(algorithm),
    medicalExample: generateMedicalExample(algorithm),
    fallbackExample: generateFallbackExample(algorithm),
    whyItFits: generateWhyItFits(algorithm)
  };
}
```

**设计优势**：
- **数据分离**：事实信息（手填）与叙事内容（生成）分离
- **一致性**：所有算法的教学结构保持统一
- **可维护性**：修改叙事模板即可批量更新所有算法说明

### 3.2 组件化设计

#### 页面区块组件

应用采用 **Section-based 架构**，每个功能区域封装为独立组件：

1. **HeroSection（英雄区）**
   - 功能：首屏展示、快捷搜索、关系图入口
   - 特点：吸引用户注意力，提供快速导航

2. **ConceptSection（概念区）**
   - 功能：解释机器学习、深度学习、神经网络的区别
   - 特点：静态教学内容，帮助初学者建立认知框架

3. **RoadmapSection（路线区）**
   - 功能：展示结构化学习路径（初级、中级、高级、医疗专项）
   - 特点：可交互的路径图，点击算法直接打开详情

4. **AlgorithmSection（算法展示区）**
   - 功能：算法卡片网格、分类筛选、搜索、对比
   - 特点：项目核心区域，整合多种交互功能

5. **ProgressPanel（进度面板）**
   - 功能：统计学习进度、显示已完成算法
   - 特点：LocalStorage 持久化，激励学习完成

#### Modal 组件系统

项目实现了**多层级 Modal 管理**：

- **AlgorithmModal**：算法详情主弹窗（支持前后切换）
- **SketchModal**：手绘图全屏查看（基于 Excalidraw）
- **ComparisonModal**：多算法对比（最多 4 个）
- **AlgorithmGraphModal**：算法关系图谱（基于 ReactFlow）
- **KeyboardHelpModal**：快捷键帮助

**设计要点**：
- 使用 `isOpen` + `onClose` 模式统一管理
- 支持 ESC 键关闭
- 阻止背景滚动
- 层级管理避免冲突

### 3.3 状态管理策略

#### 自定义 Hooks 架构

项目**不使用 Redux/Zustand**，而是通过自定义 Hooks 实现轻量级状态管理：

```typescript
// 进度跟踪 (useProgress.ts)
const { progress, toggleComplete, isCompleted, updateLastViewed } = useProgress();
// 存储在 LocalStorage，包含：已完成算法、最后查看时间、学习统计

// 算法对比 (useComparison.ts)
const { comparisonList, addToComparison, removeFromComparison,
        isInComparison, canAddMore, canStartComparison } = useComparison();
// 限制最多 4 个，提供完整的增删查逻辑

// 主题管理 (useTheme.ts)
useTheme();  // 自动读取系统偏好，存储在 LocalStorage

// 全局快捷键 (useGlobalKeyboard.ts)
useGlobalKeyboard({ onShowHelp: () => {...} });
// 监听 ? 键显示帮助
```

**优势**：
- **去中心化**：每个功能独立封装，避免全局状态污染
- **类型安全**：完全基于 TypeScript，IDE 自动补全
- **易测试**：Hooks 可单独测试（见 tests/useProgress.test.ts）

## 九、技术亮点与创新

### 9.1 叙事生成器（Narrative Builder）

**问题背景**：
- 100+ 算法需要统一的教学叙事
- 手工编写重复性高、一致性难保证
- 医疗场景示例需要专业性

**解决方案**：
```typescript
// 根据算法元数据自动生成 4 类叙事内容
function buildNarrative(algorithm: BaseAlgorithmItem): NarrativeFields {
  return {
    plainExplanation: `想象${algorithm.intuition}...`,
    medicalExample: `在医疗领域，${algorithm.name}常用于...`,
    fallbackExample: `类比到生活场景，${algorithm.name}就像...`,
    whyItFits: `${algorithm.name}适用于${algorithm.whenToUse[0]}，因为...`
  };
}
```

**价值**：
- 减少 80% 的文档编写工作量
- 确保所有算法的教学结构一致
- 支持批量更新（修改模板即可）

### 9.2 力导向图布局算法

**挑战**：
- ReactFlow 默认布局无法体现算法类别分簇
- 手工布局 100+ 节点不可行
- 需要动态适配节点增删

**创新点**：
```typescript
// clusterLayout.ts - 自定义力导向布局
export function clusterLayout(nodes, edges, options) {
  // 1. 按类别分配初始位置（圆形分布）
  const clusterCenters = calculateClusterCenters(categories);

  // 2. 力模拟迭代
  for (let i = 0; i < 300; i++) {
    applyClusterForce();    // 同类别吸引
    applyRepulsionForce();  // 不同类别排斥
    applyEdgeForce();       // 有连接的节点靠近
    applyCollisionForce();  // 防止重叠
  }

  // 3. 边界约束
  constrainToBounds(canvas);

  return { nodes, edges };
}
```

**效果**：
- 相关算法自然聚簇（如"决策树家族"紧密排列）
- 演化关系通过距离体现（如"线性回归→逻辑回归"相邻）
- 视觉清晰,易于探索

## 附录

### A. 技术选型对比

**为什么选择 React 而非 Vue？**
- TypeScript 支持更成熟
- 生态更丰富（ReactFlow、Excalidraw 等）
- 社区资源更多

**为什么选择 Vite 而非 Webpack？**
- 开发启动速度快（<1s vs 数秒）
- 热更新几乎即时
- 配置简单，开箱即用

**为什么不使用状态管理库（Redux/Zustand）？**
- 应用规模不大，自定义 Hooks 足够
- 避免过度设计，降低学习成本
- LocalStorage 直接持久化，无需额外抽象
